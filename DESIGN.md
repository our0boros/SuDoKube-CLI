# SuDoKube — 3D 数独立方体游戏设计文档

## 1. 项目概述

SuDoKube 是一款基于 **Bevy 0.18** 的 3D 数独游戏。玩家在由 6 个正方形面组成的立方体表面解题：每个面都是标准 9×9 数独，相邻面共享完整的边和角点。游戏提供深色/浅色主题、提示、撤回、自动下一局、历史记录与本地存档等功能。

## 2. 核心概念与规则

### 2.1 立方体表面网格

假设立方体边长为 9 个单元格，使用三维坐标 `(x, y, z)`，其中每个坐标取 `0..9`，且至少有一个坐标为 `0` 或 `8`。这样可唯一表示立方体表面的所有格子，并自然处理面共享：

- **8 个角点**：三个坐标均为 `0` 或 `8`。
- **12 条边**：两个坐标固定为 `0` 或 `8`，第三个坐标在 `1..8`。
- **6 个面**：一个坐标固定为 `0` 或 `8`，另两个坐标在 `0..9`。

独立格子总数：

```
8 角点 + 12 边 × 7 非角点 + 6 面 × 7×7 内部点 = 8 + 84 + 294 = 386
```

### 2.2 数独约束

- 每个 9×9 面内部满足标准数独规则：每行、每列、每个 3×3 宫格均含 1..9 不重复。
- 共享边：两个相邻面的对应边行/列完全一致，因此边上的数字自然同时满足两个面的行/列约束。
- 共享角点：三个相邻面的角格是同一数字。

### 2.3 坐标到面的映射

立方体六个面的法向与主坐标轴对应，每个面用一个 `(axis, sign)` 标识：

| 面 | 法向 | 固定坐标 | 局部坐标 (u, v) |
|---|---|---|---|
| Right  | +X | x = 8 | (y, z) |
| Left   | -X | x = 0 | (z, y) |
| Top    | +Y | y = 8 | (x, z) |
| Bottom | -Y | y = 0 | (z, x) |
| Front  | +Z | z = 8 | (x, y) |
| Back   | -Z | z = 0 | (y, x) |

局部坐标 `(u, v)` 范围均为 `0..9`，方向选择保证展开时相邻面的边方向一致。

## 4. 数据模型

### 4.1 `CubeCoord`

```rust
pub struct CubeCoord { pub x: u8, pub y: u8, pub z: u8 }
```

仅表示表面坐标（至少一维为 0 或 8）。提供与 `FaceCoord { face: Face, u: u8, v: u8 }` 的双向转换。

### 4.2 `CubeGrid`

```rust
pub struct CubeGrid {
    pub cells: HashMap<CubeCoord, Cell>,
}

pub struct Cell {
    pub answer: u8,          // 题解数字
    pub given: bool,         // 是否题目给定
    pub user_value: Option<u8>,
    pub notes: [bool; 9],
}
```

### 4.3 `Face`

```rust
pub enum Face { Right, Left, Top, Bottom, Front, Back }
```

每个 `Face` 提供 `iter_cells()` 返回该面所有 `FaceCoord`，以及 `neighbors()` 返回四条共享边对应的相邻面信息。

## 5. 题解生成算法（波函数坍缩）

目标：生成一个满足所有立方体数独约束的完整题解。

### 5.1 算法流程

1. **初始化波函数**：每个格子候选集为 `{1..9}`。
2. **坍缩 8 个顶点**：
   - 每个顶点同时属于 3 个面的角宫。
   - 优先处理熵最小的顶点（初始时所有顶点熵相同），随机选择一个候选数字。
   - 通过约束传播更新与该顶点相连的三条边及三个面的相关格子候选集。
3. **坍缩 12 条边**：
   - 每条边包含 7 个非角格子，同时受两个面的行/列/宫约束。
   - 按当前熵升序依次坍缩，每次选取候选数最少的格子；若熵相同则随机。
   - 每次赋值后立即约束传播；若出现候选集为空则回退（回溯）。
4. **坍缩 6 个面内部**：
   - 每个面剩余 49 个内部格子，按标准数独 WFC 方式填充。
   - 同样使用 MRV（最小剩余值）启发式 + 约束传播 + 回溯。
5. **验证**：最终检查每个面的行、列、宫是否均为 1..9。

### 5.2 约束传播

- 当格子 `c` 被赋值为 `v` 后，与 `c` 在同一面且同行、同列、同宫的格子候选集移除 `v`。
- 由于边/角共享，传播会自动跨面：一个面上某行被赋值后，共享该行的相邻面对应行同步更新。

### 5.3 回溯策略

使用递归 + 副本回滚：每次选择格子前复制当前候选状态。若后续失败则恢复副本并尝试下一个候选。顶点/边阶段规模小，回溯开销可接受。

## 6. 谜题生成

在完整题解基础上：

1. 复制题解作为答案。
2. 将所有格子标记为 `given = true`。
3. 随机打乱格子顺序，逐个尝试隐藏数字。
4. 每次隐藏后使用求解器验证是否仍有唯一解。
5. 重复直到达到目标隐藏数量（默认约 45%，可配置难度）。

求解器采用与生成器相同的约束传播 + 回溯，统计解的数量（上限 2 个以加速）。

## 7. 渲染方案

### 7.1 3D 立方体

- 使用 6 个独立平面作为面，放置在距离中心 `4` 个单位的位置（格子尺寸为 1），构成边长为 9 的立方体。
- 每个面由 81 个可交互的平面网格单元组成，使用 `Mesh2d`/`Mesh3d` + 独立材质。
- 文字使用 Bevy 的 `Text2d` 或 `Text3d`（通过 `bevy_text` 的 3D  billboard 方案），每个数字作为一个子实体吸附到对应格子中心。

### 7.2 高亮与背景加深

每个格子有一个背景材质组件。根据当前选中状态切换材质颜色：

- **选中数字高亮**：与选中数字相同值的格子使用高亮色。
- **相关区域加深**：选中格子的同宫、同行、同列以及通过共享边/角扩展到相邻面的对应行/列使用加深色。
- **错误提示**：用户输入与答案冲突的格子显示红色。

## 8. 交互设计

### 8.1 鼠标/触摸

- **左键点击格子**：选中该格子。
- **点击数字按钮**或按 `1..9`：向选中格子填入数字。
- 再次点击已填数字的格子：清除。

### 8.2 键盘

- `1..9`：填入数字。
- `Backspace` / `Delete`：清除。
- `H`：提示（填入当前选中格子的正确答案）。
- `Ctrl+Z` / `Z`：撤回。
- `Esc`：打开/关闭历史记录面板。

## 9. UI 设计

### 9.1 HUD 布局

- 顶部：计时器、当前难度、主题切换按钮。
- 右侧（或底部）：数字盘 `1..9`、提示、撤回、放弃、自动下一局开关。
- 中央：3D 立方体渲染区。

### 9.2 历史记录面板

按 `Esc` 呼出，显示本地保存的历史对局列表：

- 日期/时间
- 用时
- 难度
- 完成状态
- 可点击“复盘”重新加载该题解（可选）。

## 10. 持久化

使用 `rusqlite` 在本地 `sudokube.db` 中保存：

```sql
CREATE TABLE IF NOT EXISTS games (
    id INTEGER PRIMARY KEY,
    started_at TEXT NOT NULL,
    finished_at TEXT,
    elapsed_seconds INTEGER,
    difficulty TEXT,
    completed BOOLEAN,
    answer TEXT NOT NULL,      -- 题解序列化
    puzzle TEXT NOT NULL,      -- 谜题序列化
    moves TEXT                 -- 用户操作历史 JSON
);
```

序列化格式：按固定顺序将 386 个格子的数字拼接为字符串，空值用 `0` 表示。

## 11. 实现路线图

1. **基础框架**：模块拆分、`CubeCoord` / `Face` / `CubeGrid` 数据模型。
2. **生成器**：WFC 顶点→边→面生成完整题解。
3. **谜题生成器**：挖空 + 唯一解验证。
4. **Bevy 基础**：3D 场景、相机、灯光、空白立方体渲染。
5. **格子渲染**：生成 6 个面 386 个格子实体与数字文本。
6. **输入与选中**：射线拾取、数字填入、冲突检测。
7. **高亮系统**：同数字、行列宫、跨面扩展背景色。
8. **UI 控件**：数字盘、提示、撤回、放弃、自动下一局、主题切换。
9. **历史记录与存档**：SQLite 读写 + ESC 面板。
10. **整合测试**：编译修复、流程验证、性能调优。
